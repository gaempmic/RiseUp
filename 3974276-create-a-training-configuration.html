Create a training > Configuration

How to configure a training? How to make it visible in the catalog? Allow registrations? Link a certificate / assessment?
Written by Claire Philippe
Updated over a week ago

When creating a training course, the following information must be completed:

  • General information

  • Detailed information (optional)

  • Configuration (optional)

  • Information about the organization (optional)

  • Customized fields (optional)

This article focus on the Configuration section.


Default training configuration

This section is already completed with the default configuration, hence the qualification as Optional. At the very least, you should check that the default configuration is actually appropriate to your organization.

Below you will find the Default Configuration that will be applied, if you don’t modify this section.

Screen showing the Training Configuration default configuration



Visible

By default, this box is checked.

This box allows you to make the training visible or invisible in the Catalog for Multi-scope Training (in the Public Catalog for Public Training) for all users.

If the box is checked, the training is visible in the Catalog for all users (obviously only after the training has been published). If the box is not checked, the training is no longer visible in the catalog, but is only visible in the training management interface for Administrators (internal and/or partner) and the Designer ( > Training > View)


Sequential

By default, this box is unchecked.

This box allows you to make the training sequential or non-sequential. If the “sequential” box is checked, the learner will have to take the training modules and stages in the order in which they appear. The learner will have to finish one module before moving on to the next and within each module finish the previous stage before being able to begin the next stage. If the “sequential” box is not checked, the learner will be able to take the modules and training stages in any order they wish. This “sequential” button is also in the training path.

Dedicated page

By default, this box is unchecked.

The dedicated page makes it possible to communicate only the Training Description Sheet to people outside the company (customer, prospect or partners). If this box is checked, a URL link will be generated. It will therefore be possible for the Administrator or Designer to share this URL link with an external person. The external person will only have access to the training description sheet without needing access to this Training.


Access path to retrieve the URL for the dedicated page: Side Panel > Training > View > three dots icon > dedicated page URL.



Create a board on the community page

By default, this box is unchecked.

The Community is a space where learners can discuss among themselves. This space can be accessed at any time by clicking the Community tab in the top banner of your screen (see below) as long as the Community has been activated by the Administrator in the platform configuration (⚙️).

By default, the Create a board on the community page box is not checked in the Configuration section of the training being created.

By checking this box, a specific forum dedicated to the training being created will be set up and made accessible to anyone registered for this training. The forum’s subject will have the same title as the training created.


Send legal documents

By default, this box is checked.

By default, the Send legal documents box is checked, which means that the 3 legal documents related to this training, Invitation to a classroom session, Attendance certificate and Training completion certificate will be generated automatically and sent to the Learners by e-mail.

For more information on theses documents see: Generate legal documents.

If the Send legal documents box is not checked, no legal documents will be e-mailed automatically to learners.

⚠️ However, these legal documents will be generated and will remain available for learners to download by accessing their Account. You can not deactivate the automatic creation of documents.

These legal documents are configured by the administrator (internal and/or partner). To get access : Side Panel > Legal Documents > Manage Templates.

More information in the article Edit legal documents templates.

Allow registrations

By default, this box is checked.

If this box is checked, it means that learner will be able to ask to be registered for the training by accessing it through the Training Catalog.

A Sign up button then appears in the training description sheet, which learners can access in the Catalog.

In the default configuration (see the point below), learners will not automatically be able to take the training immediately after clicking this Registration button, as they must wait for approval by the Administrator and/or Designer and/or their “Group Manager(s)” to actually be registered for the training and be able to take it.

If this box is unchecked, learners will not be able to ask to be registered for the training from the Catalog. So the Registration button is not displayed in the training description sheet. Only an administrator or designer of the training or, if applicable, their Group Manager(s) (if they have been given such permission - see Side Panel > Groups) can register learners for this training.


Deactivate approval of registrations for training in the catalog

By default, this box is unchecked.

This means that when Learners click on the Sign up button in the Training description sheet, they will have to wait for Administrative Approval of their registration request (approval by an Administrator, a Training Designer or a Group Manager, as applicable).

If this box is checked, learners will immediately be registered for the training following their training registration request from the Catalog.


Allow users to unregister from training

By default, this box is unchecked.

If this box is unchecked, users don’t have the option to unregister automatically from training.

If this box is checked, users can unregister at any time during the training, without administrative approval. A window is displayed asking users to confirm their choice to unregister, in which the reason for cancelling can be given (not a required field). The administrator simply receives an e-mail notifying unregistration.

Users must pass the quizzes to complete training

By default, this box is unchecked

You can create quizzes in a training course. For each quiz, it is possible to define a minimum score to pass your stage.

If this box is unchecked, a learner who has not achieved the minimum score will still be able to complete the training, but will not be able to receive the occupational certification in the default configuration (it is possible to change the default configuration to issue the certificate even if the learner has not successfully completed the training – see section below on certificates).

If this box is checked, a user who has not achieved the minimum score in the quiz will not be able to complete the training and consequently will not obtain a training completion certificate and, as a result, will not be able to receive the certificate for certified training.

Training availability period – continuous or in periods

By default, the All the time is activated.

The training is available:

  • All the time: the training is available “all the time”, which means that the training is accessible all the time, once the learners are registered for the training.

  • In periods: The training is available “in periods”, which means that the training is only available over a specified period. When the period ends, it is possible to add a new period, so that the training is once again visible to the learners. Note that once the period has been approved, it is no longer possible to change it. The training must be duplicated to be able to change this period.

⚠️Training configured in periods cannot be included in a training path.


Configuration of training assessments (for each platform)

By default, no assessment is associated with the training being created.

For Multi-scope Training, Assessments are configured Platform-by-Platform.
There will then appear dynamically as many Assessment Configuration blocks as there are Platforms defined in the Scope. The Platform name is given in brackets.

It is possible to link assessments to a training course and to use the platform’s default assessment templates.

It is also possible to create your own assessment templates before creating training. Only Administrators (internal and partner) can create assessment templates Side Panel > Assessments > Manage templates.

For more information, see Set up live / delayed evaluations.

There are 2 types of assessments:

  • Live assessments (on-the-spot evaluation): The live assessment is generated immediately the learner has completed the training.

  • Delayed assessments: For the delayed assessment, it is possible to define a number of days after which the assessment will be generated.

It is possible to generate both assessments, or a single assessment.


Associating a certificate with training (for each platform)

It is possible to associate a training course with a Certificate previously created by an Administrator (internal or partner) : Side Panel > Certificates > Create.

See Article Create a certificate template.

By default, the Certificate associated with the training will only be issued to the learner if the training has been successfully completed, but this can be overridden and the Certificate always issued to the learner as soon the training is completed, by checking the box provided.

Once the certificate is issued to the learner, the certificate appears on the learner’s dashboard.
