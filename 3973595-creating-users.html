Creating users

Last updated on 08/03/2020
Written by Kevin Legros
Updated over a week ago

To create a user, click on the Side menu > Users > Create.

Fields that need to be completed in the User Creation Sheet are divided into 2 sections:

  • General information

  • Customized fields

General information

  • Username*: this is the user’s login name that will be used to connect to the platform (without SSO) combined with the password. It must be unique and contain no “blank” spaces. An error message warns you if a user already has the same login name. This is the only truly required field when creating a user. It is possible to use the user’s e-mail address as Username.

  • Last name

  • First name

  • E-mail: the e-mail address is required when the password is not specified, i.e. when Create a password is selected.

  • Phone number

  • Language: the chosen language will determine the language of the E-mails and legal documents that will be sent/generated (English will be the chosen language by default, unless otherwise stated)

  • Internal ID: this is an internal identification number

  • Password: the choice is given between:

> Create a password: in this case, a valid user e-mail address must be entered, enabling the user to receive a login request and create their own password when they log in to the platform for the first time.

> Specify the password: in this case, the administrator who creates the user imposes their first password, but can ask the user to change their password when they log in for the first time by checking the dedidacted box.

  • One Role to choose among 5 options :

> Learner

> Administrator

> Creator/Designer

> Community manager

> Session Instructor (*)

(*) The Session Instructor role must be reserved for people, outside of the company, leading classroom sessions or virtual classes. By construction, a User with a "Session Instructor role" is someone outside the company and therefore cannot follow training courses himself.

It is possible later to designate other users (Administrator, Creator/Designer, Learner, Community Manager) who are by construction internal to the company, as Session Instructors of such or such classroom session or virtual classroom. This is referred to as the “Host” attribute.

  • Date of suspension : useful to fill in this field when the user, for example, has a mission limited in time or is on a fixed-term contract

  • Biography: it's a free field

  • Profile photo : it is possible to upload a profile photo by clicking on the image
